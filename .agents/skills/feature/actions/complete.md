# Complete Action

1. Confirm that the active goals are met and that test and build results are current.
2. Update `context/current-feature.md`: set status to `Complete` and retain the goals and notes as the completion record. Append a concise completed-feature entry to `context/feature-history.md`; never revise prior history entries.
3. Before any commit, merge, push, deployment, or branch deletion, state the exact target and expected effect and obtain explicit user approval.
4. After approval, perform only the approved actions. If merging is approved, use the repository's default branch; if deleting a branch is approved, identify its exact local or remote name first.
5. Report the resulting commit, branch, and validation status. Never reset the working file or delete a branch by default.
