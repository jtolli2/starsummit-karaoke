# Review Action

1. Read `context/current-feature.md` and review the feature diff against every goal.
2. Identify completed goals, missing behavior, bugs, scope creep, and maintainability concerns.
3. Check karaoke-specific boundaries: no client secret exposure, safe PocketBase authorization and validation, resilient real-time queue handling, and no guest path that can take tablet-only playback control.
4. Verify that tests and build evidence match the changed code.
5. Give a verdict: ready to complete, ready with noted follow-ups, or needs changes.
